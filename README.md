# Experimental Physics: Methods, Measurement, and Apparatus

An open-source course and reference for advanced undergraduates, MSc, and PhD students.

Built with [Quarto](https://quarto.org), producing both a static website and a print-quality PDF book from a single source. Every figure is generated by code in the repository — nothing is a manual screenshot.

---

## Content structure

The book has two parallel layers:

**Core chapters** (`parts/`)  
The lecturable spine: one chapter per module, divided into three tiers —
*Core* (examinable), *Going deeper* (MSc/PhD extensions), and *Bench* (pointers to Bench Notes).

**Bench Notes** (`bench-notes/`)  
Dense, single-topic reference cards: concept → governing equations → decision rule → failure modes. Modelled on JPE's *Precision Point* notes. Listed on the [Bench Notes index page](bench-notes/index.qmd).

## Building locally

### 1. Prerequisites

- [Quarto ≥ 1.5.57](https://quarto.org/docs/get-started/) — install first
- Python ≥ 3.11
- A LaTeX distribution (for PDF): [TinyTeX](https://yihui.org/tinytex/) is simplest

```bash
# Install TinyTeX via Quarto
quarto install tinytex
```

### 2. Python environment

```bash
python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 3. Citation style

The repo uses the American Physical Society (APS) CSL style.
Download it from the [Zotero CSL repository](https://www.zotero.org/styles/american-physics-society)
and place it at `csl/american-physics-society.csl`.

```bash
curl -o csl/american-physics-society.csl \
  "https://www.zotero.org/styles/american-physics-society"
```

### 4. Render

```bash
# Both HTML and PDF
quarto render

# HTML only (faster for writing)
quarto render --to html

# PDF only
quarto render --to pdf

# Live preview (HTML, auto-reloads)
quarto preview
```

Output lands in `_book/`. The PDF is `_book/Experimental-Physics.pdf`.

## Adding content

### New core chapter

1. Copy `parts/_chapter-template.qmd` into the appropriate `parts/NN-<part>/` directory.
2. Fill in the three tier sections (Core, Going deeper, Bench).
3. Add a reproducible figure (see below).
4. Add the new file to the `chapters:` list in `_quarto.yml`.

### New Bench Note

1. Copy `bench-notes/_template.qmd` and rename it.
2. Fill in the six fixed fields: Concept, Governing equations, Decision rule, Failure modes, See also, References.
3. Add a card link in `bench-notes/index.qmd`.
4. Add the file to `_quarto.yml` under the Bench Notes part.

### Reproducible figures

Every figure must be generated by a Python script in `figures/src/` or an executable Quarto code cell:

- Use `rng = np.random.default_rng(seed=42)` for any random data.
- Read from `data/`; write outputs to `figures/` (gitignored).
- Apply `plt.rcParams.update({...})` from the shared style rather than matplotlib defaults.
- Scripts should be runnable standalone: `python figures/src/my_figure.py`.

See `figures/src/chi2_fit_residuals.py` for the reference example.

## CI/CD

Every push to `main` triggers `.github/workflows/publish.yml`:

1. Installs Python dependencies (cached).
2. Installs Quarto 1.5.57 (pinned).
3. Installs TinyTeX + required LaTeX packages.
4. Renders HTML → deploys to GitHub Pages.
5. Renders PDF → uploads as a build artifact (downloadable from the Actions tab).
